**日期:** 2026-05-12
**环境:** WSL2 / Ubuntu

---

## 问题: `mapp.api.weibo.cn/fx/` 微博快照链接无法剪藏

**URL 格式:** `https://mapp.api.weibo.cn/fx/{hash}.html`

### 现象
1. **URL 路由错误**: 该 URL 未匹配 WeiboAdapter（适配器只认 `m.weibo.cn` / `weibo.com`），fallback 到 GenericWebAdapter
2. **反爬验证页**: Generic 适配器抓取到的是「Sina Visitor System」验证页，非真实内容
3. **空 Markdown**: 最终产出内容为空，只有元数据占位

### 建议
1. WeiboAdapter URL 模式增加 `mapp.api.weibo.cn`
2. `mapp.api.weibo.cn/fx/` 这类链接可能是微博快照/分享链接，需要特殊处理：
   - 可能需要设置特定的 User-Agent 或 Cookie
   - 或解析 redirect/iframe 中的真实内容 URL

### 备注
即使 URL 路由正确匹配到 WeiboAdapter，WeiboAdapter 当前通过 `m.weibo.cn/statuses/show?id={mid}` API 获取内容，但 `mapp.api.weibo.cn/fx/` 链接没有直接的 mid，可能需要先请求该 URL 获取跳转或解析快照中的 mid。
